# Visual Banners and Page Header Styles

Visual banners and page header styles help page readers notice the selected classification level.

Classify is designed to make classification visible in the page header or classification area, rather than hiding it in metadata alone.

## What the banner communicates

The visual banner communicates:

- The selected classification level.
- The visual importance of that level.
- The handling guidance users should consider before reading, editing or sharing the page.

## Expected page header style options

Known expected options are:

| Option | Description |
| --- | --- |
| None | No background. |
| Color Band | Uses the background color. |
| Upload Background | Allows the admin to upload a custom background. |
| Predefined Background | Allows the admin to select an existing predefined background. |

## Classification and permissions

A visual banner does not replace Confluence permissions. It helps users understand the intended sensitivity and handling rules for the page.

:::note
Use the page header style to support visibility. Use Confluence permissions to control access.
:::

## Notes for admins

Choose visual styles that are clear without being distracting.

For sensitive levels such as **Confidential** and **Restricted**, use a style that is visible enough for readers to notice before they share or reuse the content.
