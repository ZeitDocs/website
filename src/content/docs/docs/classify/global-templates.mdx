The **Classification Templates** screen (under **Apps → Classification
Templates**) is where a Confluence administrator defines classification **once
for the whole site** and rolls it out to many spaces consistently. This is the
recommended way to keep every space using the same classification levels,
colours, labels, and restrictions.

## What a global template is

A global template is a named, reusable set of **classification levels** — the
same levels described in
[Space Settings](/docs/classify/space-settings/#what-is-in-a-classification-level)
(name, colour, description, view and edit groups, label, page header style and
image).

When a space admin applies a template, the levels are **deep-copied** into that
space. The space then runs independently — later edits to the template do not
retroactively change spaces that already applied it (they re-apply to pick up
changes).

## The template library

The screen lists all global templates. Classify ships with **6 predefined
templates** (for example *Enterprise Standard* and *Simple Business*) that are
seeded automatically the first time you open the screen. You can use these as
they are, edit them, or add your own.

### Creating or editing a template

1. Add a new template or open an existing one.
2. Give it a **name** and **description**.
3. Add **classification levels** — the same fields as space levels.
4. Save.

### Enabling or disabling a template

Each template has an **enabled** toggle. Only **enabled** templates appear to
space admins in the byline onboarding and the "Add from template" picker. Disable
a template to retire it without deleting it.

## Applying a template to spaces

From a template you can push it to many spaces at once via **Apply to Spaces**:

1. Open the **Apply to Spaces** panel for the template.
2. Search and select the spaces (paginated 25 per page).
   - Spaces already linked to this template are **pre-selected, greyed out, and marked "Already linked"** — you cannot double-apply.
   - The selection count and Apply button reflect only **new** selections.
3. Click **Apply**.
   - If some spaces already have a configuration, you see a **conflict prompt** — confirm to **override** them, or cancel.
4. Classify applies the levels and updates the template's linked-spaces list.

### Spaces that get skipped

Any space **blocked by Space Access Control** is skipped automatically — it is
never written, even if you selected it. The result shows an info message such as:

> *"N skipped — blocked by Space Access Control: …"*

This is a safety guard: an admin cannot accidentally classify a space that policy
forbids. See [Space Access Control](/docs/classify/space-access-control/).

## Seeing which spaces use a template

Each template shows its **linked spaces** as pills (up to 5). If more than five
spaces are linked, a **"+N more"** link opens a dialog listing them all.

## Manage access

A **Manage access** button at the bottom of the templates list opens **Space
Access Control**, where you decide which spaces are allowed to use classification
at all. See [Space Access Control](/docs/classify/space-access-control/).

## Tips

- **Edit the predefined templates** rather than starting from scratch — they cover common classification schemes.
- **Disable instead of delete** when retiring a scheme, so existing spaces keep working.
- Changing a template does **not** auto-update spaces that already applied it — re-apply the template (with override) to push changes out.
